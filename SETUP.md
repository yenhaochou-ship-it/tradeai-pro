# 部署設定（Vercel 環境變數）

到 Vercel 專案 → Settings → Environment Variables 設定：

| 變數 | 說明 |
|---|---|
| `SINOPAC_BACKEND_URL` | 原本就有，後端(Railway)的網址，不變。 |
| `BACKEND_API_KEY` | **跟後端(tradeai-backend)的 `BACKEND_API_KEY` 設成完全一樣的值**。這支proxy會用這個值附帶 `X-API-Key` header 打給後端，後端才會放行。這個值完全是伺服器端的環境變數，不會出現在瀏覽器/打包出來的前端程式碼裡。 |
| `APP_PASSWORD` | 自己取一個密碼（給你自己/家人用的，不需要很複雜，但別用「1234」之類太容易猜的）。這是擋住「外人知道你的Vercel網址、直接打/api/sinopac控制你的交易機器人」的關鍵——這支proxy本身是公開在網路上的，不管你前端畫面長什麼樣子都一樣。沒設定的話這層檢查會跳過(方便本機測試)，但正式使用前請務必設定。 |

設定好以上兩個值之後，重新部署（Vercel 改環境變數後通常需要重新觸發一次部署才會生效）。

## 使用者體感上的變化

- 開啟網站第一次會看到一個「請輸入應用程式密碼」的畫面——輸入你在 `APP_PASSWORD` 設定的密碼即可，
  之後這台裝置/這個瀏覽器不會再問（除非清掉瀏覽器資料，或你之後改了密碼）。
- 永豐帳戶的「連接真實帳戶」不再需要輸入 API Key/Secret Key——這兩個值已經搬到後端(Railway)的
  環境變數，前端開啟後會自動幫你連線，完全不用手動輸入或記住任何永豐金鑰。
- 已移除原本沒有被用到的「AI 問答」後端代理(`api/anthropic.js`)——目前畫面上沒有任何地方
  在呼叫它，移除後可以少一個沒人用、卻仍然公開在網路上可被濫用的端點(用你的 Anthropic 額度)。
  如果之後想做這個功能，再重新接上即可，邏輯本身（金鑰只留在伺服器端）是對的，只是先移除沒在用的部分。
